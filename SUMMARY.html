End-of-Turn Detection — Summary

Plivo ML Assignment — predicting whether a pause in user speech is the end of turn (EOT) or just a hold, using only audio available before the pause.

Approach

Built a feature-based classifier rather than anything pretrained (not allowed by the rules anyway). For every pause, features are computed only from audio between turn start and pause_start — never after, so the causality rule is respected everywhere.

Features used (all causal):

  • Pitch and pitch slope over the trailing window of speech before the pause
  • RMS energy and energy trajectory
  • Voiced ratio
  • Pause position in the turn (pause_index) and time since turn start
  • Pause-history features: number of prior pauses, duration of the previous pause, running mean/max of prior pause durations — all use only pauses that already happened

Model: GradientBoostingClassifier (scikit-learn), trained on English + Hindi data combined. Also tried HistGradientBoostingClassifier with the same features — it underperformed on cross-validation and was dropped.

Why combined-language training

Ran a controlled experiment: trained on English only, tested on Hindi (a stand-in for an unseen language). This scored AUC 0.616 and 850ms delay on Hindi — no better than the silence-only baseline. Training on English+Hindi together generalizes much better, so the submitted model is trained on both.

Iteration log (condensed from RUNLOG.md)

  1. Baseline — silence-only, AUC ~0.50 both languages, confirms no real signal without features.
  2. Added prosodic features (pitch, energy, voiced ratio) + GradientBoostingClassifier trained on English+Hindi combined. CV AUC 0.631, but in-sample numbers (0.98+) turned out to be overfitting.
  3. Cross-language check — English-only training tested on Hindi scored no better than baseline, confirming combined-language training is necessary.
  4. Added pause-history features and compared GBM vs HistGBM — GBM won on CV AUC (0.629 vs 0.611).
  5. Built an honest held-out evaluation — 40 turns never touch training, giving a trustworthy AUC of 0.682, well above baseline and confirming the model isn't just memorizing.
  6. Final model retrained on all data (English+Hindi, no exclusions) for submission, since the held-out check already validated generalization.

Human vs. AI coding agent

AI agent Helped generate the initial project structure, feature extraction code, training pipeline, prediction script, and some boilerplate code to speed up development.

Human I decided which features to keep, compared different models, and checked whether the results were actually reliable. After noticing that the very high training scores were caused by overfitting, I created a separate held-out evaluation to measure real performance. I also ran an English-only to Hindi experiment to understand cross-language performance, added pause-history features, tuned the Gradient Boosting model, compared it with HistGradientBoosting, and selected the better model based on validation results.

The coding agent helped with implementation, but the experiments, model decisions, evaluation strategy, and final improvements were based on my own analysis and observations.
